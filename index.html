Propstack V1 Request Tester

Eigenstaendige HTML-Seite zum Testen von Propstack-V1-API-Requests. Du kannst API-Key, URL, Request-Methode und Body direkt eingeben und den Request aus dem Browser senden.

Request

Standard-Header X-Api-Key und Accept: application/json werden automatisch gesetzt.

Beispiel: https://api.propstack.de/v1/client_properties
Bei JSON wird automatisch Content-Type: application/json gesetzt. Bei GET-Requests wird ein eingegebener Body vom Browser ignoriert.
Wenn Propstack fuer den angesprochenen Endpoint keine CORS-Freigabe liefert, blockiert der Browser den Request trotz korrekter Daten. In dem Fall brauchst du einen lokalen Proxy oder einen API-Client wie Postman/Insomnia.

Response

Status, Laufzeit, Header und Response-Body werden hier angezeigt.

Response Body

Hinweise

Kleine Helfer, damit die Seite im Alltag schneller nutzbar ist.

Lokale Speicherung API-Key, URL, Methode und Body werden in localStorage gespeichert, damit du nach einem Reload weitermachen kannst.
Direktes Oeffnen Die Datei kann direkt im Browser geoeffnet werden. Falls dein Browser bei file:// restriktiv ist, serviere sie kurz lokal, z. B. ueber einen kleinen HTTP-Server.
Propstack Header Die Seite nutzt den ueblichen Header X-Api-Key. Zusaeztliche Custom-Header sind in dieser Version bewusst nicht enthalten, damit die Seite eigenstaendig und kompakt bleibt.
